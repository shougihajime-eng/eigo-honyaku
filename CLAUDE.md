## 🔒 永続ルール（絶対遵守・毎回これを守る）

このプロジェクトの**テロップ／字幕**を扱うすべての作業で、ユーザーから明示の指示が無くても**最初から以下を守る**。
コード変更・プリセット追加・新規画面追加・既定値変更・翻訳プロンプト調整など、テロップに触れる全作業で適用。

### テロップ配置の永続ルール

1. **黒帯部分（画面下端の暗い領域）に収まる場合は、必ずそこに配置する**
2. **黒帯が無い場合は、画面下部の見やすい位置に配置する**（下端から余白 4〜8% 目安）
3. **中央配置は禁止**（読みづらい・画面を遮る）。`align: "middle"` を既定値にしない／UIで目立たせない
4. **見やすさを最優先する**（フォントサイズ・コントラスト・縁取り・帯背景でハッキリ見せる）
5. 人物の顔・将棋盤・重要な局面に重ねない
6. 背景と同化する色は禁止（白背景に白文字、黒背景に黒文字 など）

→ コード上の既定値は `lib/telop/defaults.ts` の `DEFAULT_STYLE`（黒帯＋白文字・下部配置）。
→ 新しい既定値・新規プリセットを追加する場合も、上記6点を満たすこと。

### 翻訳品質の永続ルール（最優先）

1. **「直訳」ではなく「ネイティブが見ても自然な英語」を最優先**にする
2. **動画として自然な英語**（YouTube字幕として読みやすい話し言葉のテンポ）
3. **将棋用語・配信特有の言葉も意味が伝わる訳**にする（辞書語は守るが、訳全体としては自然さ優先）
4. **同じ日本語に対して訳がブレない**よう、プロンプトは決定論的に書く（recent context・温度設定を低めに）
5. **機械翻訳っぽい言い回しは禁止**（"Now then,..." "It is..." など定型ロボ調はNG）
6. 段位・棋士名・JSF表記は維持しつつ、地の文は自然な英語に

### UI・デザインの永続ルール

1. **使いたくなるデザイン**：シンプルで見やすいまま、少し高級感とワクワク感を足す（ガラス感・控えめなグラデーション・滑らかなアニメ）
2. **ボタンは大きく**（最低タップ領域 48px、主要ボタンは 56px 以上が理想）
3. **文字は大きめ**（本文 16px 以上、見出しは 24px 以上、スマホで読める）
4. **余白を整理**（情報を詰め込まず、ひと呼吸入る間隔）
5. **iPhone / Android / iPad / PC すべてで崩れない**（モバイルファースト、セーフエリア対応）
6. **派手なだけで使いにくい UI は禁止**

### 機能の永続ルール

1. **翻訳後の修正が直感的にできる**（その場で直接編集・即反映）
2. **複数言語対応に拡張しやすい構成**を意識する（言語コード切替で増やせる作り）
3. **処理速度を重視**（無駄なAPI呼び出しを避ける・キャッシュを使う）
4. **小学生でも感覚的に使えるくらい分かりやすく**
5. 「なんとなく動く」ではなく**実際の編集でストレスなく使える完成度**

### 秒読み演出の永続ルール

- 「いち、に、さん…」の音声系表現ではなく、**数字（1, 2, 3, 4...）で大きく見せる**
- 直感的に見える形を優先（10〜6 白／5〜3 黄／2〜1 赤 など段階色）

### 禁止事項

- 見た目だけ派手で使いにくい UI
- 翻訳が毎回ブレる仕様
- 端末によって崩れるデザイン
- 英語が機械翻訳っぽくなる状態

## 📺 最重要画面：字幕確認画面

**このアプリの心臓部**。ここがダメなら全部ダメ。

### 一瞬で確認できるべき4つの情報

1. 日本語（元の文）
2. 英語（翻訳結果）
3. 将棋用語（正しい訳になっているか）
4. 誤訳警告

### 重要用語の表示方法（見落とし防止）

- 色分け
- 強調表示
- 辞書アイコン

## 📛 翻訳の絶対ルール（将棋界専門編集者として）

これは一般翻訳ではなく **「将棋YouTube動画の海外向けローカライズ」**。
正確性を「自然な英語」より優先する。コードの翻訳プロンプト（`lib/video/translate.ts`）と辞書（`lib/shogi-dictionary.ts`）に反映済み。

### 最重要：固有人名

| 日本語 | 英語 | 禁止表記 |
|---|---|---|
| 鈴木肇 | Hajime Suzuki | 鈴木始／鈴木元／Suzuki Hajimu／その他の勝手な変換 |

→ チャンネル運営者ご本人の名前。**絶対に1文字も変えない**。
→ 辞書 (`lib/shogi-dictionary.ts`) の name カテゴリに登録済み。新しい棋士名を扱うときも同じ要領で追加する。

### 翻訳全体の方針

- 棋士名・戦法名・棋戦名・段位は **推測禁止**。確信がない場合は原文を残す
- 段位・称号（七段、名人、王座、竜王 など）は省略しない（例：「藤井名人」→ "Fujii Meijin"）
- 将棋専門用語は **日本将棋連盟の公式英語表記を優先**
- AIっぽい大げさな装飾・存在しない英単語の創作は禁止
- 意味を変える意訳は禁止。原文の温度感を保つ

### 翻訳前の固有名詞チェック（運用ルール）

将棋動画を翻訳する前に、以下を一覧化してから翻訳に進む：
1. 棋士名（漢字＋公式英語表記）
2. 戦法名（辞書にあるか）
3. 段位・称号
4. 棋戦名（竜王戦・名人戦 等の英語表記）
5. サムネ文字・タイトル

不明な固有名詞があれば **翻訳停止し、確認を求める**。
※ この「事前チェック画面」は次フェーズで実装予定（現状は翻訳後の警告で拾う）。

## 🎬 テロップ機能（追加要件 2026-05-12）

**SRT を作るだけでは不十分**。動画にそのまま乗せられる「見やすい英語テロップ」のデータも作る。
海外視聴者がスマホで一瞬で読めることが目的。

### テロップ要件（必須）

- **配置の最優先は「黒帯＋白文字」**（チャンネル統一感＋最高の視認性。`DEFAULT_STYLE` で固定）
- 黒帯に収まらない場合のみ、画面下部の見やすい位置へ配置
- **スマホ視聴**でも読める文字サイズ
- 1行が長くなりすぎないよう自動改行
- **原則2行以内**
- 1字幕あたり長すぎる英文を出さない
- 表示時間が短すぎる字幕は警告
- フォントサイズ・位置・背景・縁取りを画面で調整できる
- プレビュー画面で実際の見え方を確認できる

### 配置の禁止事項

- **画面中央**への配置（読みづらく、画面を遮る）
- **人物の顔**に重なる配置
- **将棋盤や重要な局面**を隠す配置
- **背景と同化**して読みにくい字幕（白い背景に白文字など）

→ 最優先は「**一瞬で読めること**」

### プレビューでリアルタイムに確認できること

- どう見えるか
- 小さすぎないか
- 読みやすいか
- 将棋盤を邪魔していないか

### 出力形式（最初は2種類）

1. **YouTube 用 SRT ファイル**（字幕標準形式・どこにでも貼れる）
2. **動画焼き込み用テロッププロジェクトJSON**
   - 字幕テキスト＋タイムコード＋見た目設定（フォント／色／位置／縁取り／背景）を1ファイルに保存
   - 将来 `ffmpeg drawtext` で焼き込み動画を書き出すのに使う

### スコープ

- 最初は「テロップ設定したJSONとプレビュー」までで OK
- 焼き込み動画書き出しは Phase 2（将来）
- **最初から高度な動画編集ソフトにはしない**（カット・エフェクト等は不要）

### 字幕の品質ルール（翻訳プロンプトに渡す）

字幕は「翻訳が正しい」だけではダメ。YouTube では**見やすい・読みやすい・テンポがいい**字幕でなければ意味がない。

- 1字幕は短く（半角40文字目安・絶対50文字以内）
- 自然な英語
- 句読点や改行で**読むテンポ**を作る
- 将棋専門用語は辞書通り

### 🛠 Google Cloud セットアップ手順（非エンジニア向け・1回だけ）

動画字幕機能を使うには Google Cloud の準備が必要。**5分くらいの作業**。

1. **Google アカウントでログイン**
   https://console.cloud.google.com/ を開いて Google アカウントでログイン
2. **プロジェクトを作る**
   画面上の「プロジェクトの選択」→「新しいプロジェクト」→ 名前は何でも OK（例：`eigo-honyaku`）→ 作成
3. **Speech-to-Text API を有効化**
   左メニュー「API とサービス」→「ライブラリ」→ 検索バーに `speech` → 「Cloud Speech-to-Text API」→「有効にする」
4. **お金の設定（無料枠で十分・カード登録だけ）**
   左メニュー「お支払い」→「請求先アカウントをリンク」→ クレジットカードを登録
   ※ Google Speech は **毎月60分まで無料**。5分動画なら12本／月までタダ
5. **サービスアカウントを作る**
   左メニュー「IAM と管理」→「サービスアカウント」→「サービスアカウントを作成」
   名前：`eigo-honyaku-app` → 作成 → ロール「Cloud Speech クライアント」を付与 → 完了
6. **JSON キーをダウンロード**
   作ったサービスアカウントを選択 →「キー」タブ →「鍵を追加」→「新しい鍵を作成」→ JSON 形式 → 作成
   ※ ダウンロードフォルダに JSON ファイルが入る
7. **`.env.local` に貼り付け**
   - メモ帳で `C:\Users\shoug\英語翻訳\.env.local` を開く（無ければ新規作成）
   - ダウンロードした JSON ファイルもメモ帳で開く
   - JSON の中身を **1行にして** 以下のように書く：
     ```
     ANTHROPIC_API_KEY=sk-ant-あなたのキー
     GOOGLE_APPLICATION_CREDENTIALS_JSON={"type":"service_account",...全部1行...}
     ```
   - 改行を取り除く必要は実はないが、わかりやすさのため `{...}` 全体を1行で。改行があってもアプリ側で扱う
8. **完了**
   `npm run dev` で起動して http://localhost:3000/video を開く

## 📦 クイック翻訳機能（実装済み）

YouTube運営中に「タイトル・サムネ文字・概要欄・コメント返信」などをすぐ英訳したい場面のための機能。
**動画アップロード不要・テキストだけで即翻訳**。

### 4つのモード

| モード | 用途 | プロンプト方針 |
|---|---|---|
| 自然英語 | 海外読者に自然な英語（普段使い） | 文法的に正しく、英語として滑らかに |
| YouTubeタイトル | クリックされやすい英語タイトル | 短く・インパクト・釣りすぎない |
| 将棋専門 | 解説などで用語を正確に | 海外の将棋ファンに通じる正式英訳 |
| 直訳 | 原文に忠実 | 意訳・装飾なし |

### 将棋辞書

`lib/shogi-dictionary.ts` に 100+ 語（駒・戦法・囲い・手筋・棋戦名 など）。
入力に辞書語が含まれていれば、AI に「必ずこの英訳を使え」と強制ヒントを渡す。
辞書ヒットは画面下にバッジ表示。

辞書追加は配列に1行足すだけ：
```ts
{ jp: "新しい用語", en: "English Term", category: "opening" },
```

### UI 仕様

- モード4つを上部に大きく表示（タップで切替）
- 日本語入力 → 英訳ボタン or `Ctrl + Enter`
- 結果は「コピー」ボタンで一発コピー
- スマホ縦持ちでも崩れない（max-w-3xl 中央寄せ、grid 2列→4列）
